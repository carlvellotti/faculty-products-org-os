# /templates/ — navigation map

The shared formats the team uses. If you're filling out a doc, start here.

## What's here

| Template | When to use |
|---|---|
| `customer-interview.md` | After any external user call |
| `prd-template.md` | Any feature ≥ 1 sprint of work |
| `status-update.md` | Weekly team update |

## Rule

If the template doesn't fit your case, **edit the template**, don't fork it. One canonical version per doc type.

If a doc type isn't here yet, the path is: write it once, PR it in as a template, then everyone uses it.

## Format

Templates use placeholder syntax `{{ ... }}` for things you fill in. They include guidance comments (`<!-- ... -->`) inline that you remove before publishing.
