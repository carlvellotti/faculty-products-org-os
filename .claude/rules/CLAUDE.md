# /.claude/rules/ — navigation map

Standards Claude applies automatically when working in this repo. Read these once; they shape every interaction here.

## What's here

| Rule | When it applies |
|---|---|
| `file-naming.md` | Any time you create a new file |
| `prioritization.md` | Any time you need to rank work (PRD, backlog, planning) |

## How rules work

Claude Code loads `.claude/rules/*.md` automatically when you open the repo. They're not a separate prompt — they're context Claude has at all times. Treat them like team-wide preferences a new teammate would learn in their first week.

## When to add a new rule

You catch yourself correcting Claude (or a teammate) the same way more than twice. That's a rule. Write it down, PR it.

## Rule about rules

Rules describe how the team **already** works. They're not aspirations or wish-lists. If the team isn't doing it, it's not a rule yet.

## Owner

Carl. New rules go through PR review with at least one other teammate.
